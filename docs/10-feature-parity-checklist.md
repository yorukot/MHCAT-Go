# Feature Parity Checklist

Status: Ticket config repository foundation, ticket setup/delete UI handlers, `tic`/`del` open/close side-effect handlers, runtime feature flag, and staging command-sync include gate are implemented after Platform Wave B. A repeatable slash-command UI parity audit now compares all 74 legacy slash command definitions against current Go definitions; the current report shows 44 matching Go definitions, zero UI metadata drift, and 30 legacy command names still without Go definitions. Poll Wave A/B now implements gated poll create/vote/owner-menu/result chart/text export/Excel export handlers with legacy-style visible UI and safer repository vote semantics. Low-risk utility commands `help`, `ping`, `info bot`, `info shard`, `info user`, and `info guild` are implemented and tested. `/help`, `/info bot`, `/info shard`, `/info user`, and `/info guild` now render legacy-style interfaces, with `/info shard` intentionally fixing the old empty initial embed. Read-only `/代幣查詢` is implemented behind an explicit economy-query feature flag. `/簽到` has a disabled-by-default, staging-gated first write slice with legacy-style loading/image/buttons, and `/簽到列表` is implemented as a read-only list/export path under the same gate. The economy daily reset and work payout paths have dry-run-first one-shot tools, and Mongo scheduler lease infrastructure exists, but recurring scheduler loops remain pending. `打工系統` now has a gated command-definition shell, exact legacy dashboard redirect for `新增打工事項`, legacy-style `打工介面` list/captcha/detail/start UI with an explicit atomic `work_users` start-write path, and admin setup/delete/energy writes behind explicit admin repository wiring. Read-only `/警告紀錄` is implemented behind explicit warning-history runtime and staging command-sync gates. Config-only `/警告設定` is implemented behind separate warning-settings runtime and staging command-sync gates and writes only `errors_sets`. `/翻譯` is implemented behind explicit translate runtime and staging command-sync gates. Read-only `/查看餘額` and write `/兌換` are implemented behind separate explicit runtime and staging command-sync gates. `/自動聊天頻道` and `/自動聊天頻道刪除` are implemented as config-only writes behind explicit runtime and staging command-sync gates; chatbot message runtime remains disabled. `/自動通知列表` and `/自動通知刪除` are implemented as auto-notification config-maintenance commands behind explicit runtime and staging command-sync gates; `automatic-notification` setup and scheduled sends remain disabled. `/防詐騙網址` is implemented as a config-only `good_webs` toggle behind explicit runtime and staging command-sync gates; `/詐騙網址回報` is implemented behind explicit runtime, command-sync, and webhook gates; message deletion remains disabled. `/語音包廂設置` and `/語音包廂刪除` are implemented as config-only `voice_channels` maintenance commands behind explicit runtime and staging command-sync gates; dynamic voice-room creation and locks remain disabled. `/set-log-channel` now has a gated logging-config slice with legacy-style visible UI and rollback-compatible `loggings` writes. Read-only `/扭蛋獎池查詢` is implemented behind explicit gacha runtime and staging command-sync gates; draw, inventory, and shop writes remain disabled. `/抽獎設置` now preserves the legacy disabled-command unavailable response behind explicit runtime and command-sync gates; lottery creation and component behavior remain disabled. `/統計系統查詢` now preserves the legacy static stats help embed behind explicit runtime and command-sync gates; stats channel mutations and rename worker remain disabled. `/聊天經驗` and `/語音經驗` now preserve the legacy disabled-command replacement response behind explicit runtime and command-sync gates; XP profile/rank rendering remains disabled. Config-only `聊天經驗設定`, `聊天經驗刪除`, `/驗證設置`, gated `/驗證` captcha/role/nickname flow, `/帳號需創建時數`, `/生日系統 祝福語設定`, `/生日系統 增加`, `/生日系統 是否允許管理員設定`, `/生日系統 刪除`, `/生日系統 生日列表`, and the gated member-add account-age kick/log flow are implemented behind explicit runtime and staging command-sync gates; XP accrual/rank/voice behavior and birthday notifications remain disabled. Ticket, poll, economy-query, economy sign-in, work, warning-history, warning-settings, translate, balance-query, redeem, autochat-config, auto-notification-config, anti-scam-config, anti-scam-report, voice-room config, logging-config, gacha prize-list, lottery disabled-command, stats-query, birthday-config/profile, text-XP config, voice-XP config, XP profile disabled-command, verification config, verification flow, and account-age runtime side effects remain disabled by default and must be paired with staging-only command sync before live use. Destructive moderation, recurring scheduler, ChatGPT/autochat message runtime, full automatic-notification setup/sends, full stats channels, anti-scam message deletion runtime, dynamic voice-room creation/locks, birthday delivery scheduler, and full XP remain `not-started`.
